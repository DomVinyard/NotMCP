Why notmcp

Tools without the ceremony

Most tools just call an API. You don't need a protocol for that.

Skip the server setup

Tools are just Python scripts, not running services. No ports, no processes, no Docker. Your agent runs a script, gets a result, moves on.

The contract

Simple by design

JSON in, JSON out. That's the entire protocol.

#!/usr/bin/env python3
"""
name: github-list-repos
description: List repositories for the authenticated user
credentials:
  - GITHUB_TOKEN
"""
import json, os, sys
from urllib.request import urlopen, Request

def main():
    token = os.environ["GITHUB_TOKEN"]
    
    req = Request("https://api.github.com/user/repos")
    req.add_header("Authorization", f"Bearer {token}")
    
    data = json.loads(urlopen(req).read())
    repos = [r["name"] for r in data]
    
    print(json.dumps({"repos": repos}))

if __name__ == "__main__":
    main()

FAQ

Common questions

Which AI agents are supported?

Claude Code works out of the box. Cursor requires the nightly channel (Settings → Beta → Nightly). Both auto-discover skills from ~/.claude/skills/.

Can tools use pip packages?

We recommend stdlib-only for portability, but nothing stops you. The only contract is JSON in, JSON out. If you have packages installed, use them.

How do I create a new tool?

Just ask your agent: "Create a tool for [whatever you need]." Or run notmcp create my-tool and edit the scaffolded script.

What about MCP?

MCP is great for structured tool ecosystems. notmcp is for a different problem: most tools are simple, and the protocol overhead isn't worth it.
